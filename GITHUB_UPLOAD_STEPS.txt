FITNESS V4 — GITHUB PAGES UPLOAD
================================

NEW REPOSITORY
--------------
1. Extract fitness_v4_anatomy_bilingual_cloud_sync_preview.zip.
2. Open the extracted fitness_v4 folder.
3. In GitHub, create or open a repository, for example: fitness-v4.
4. Select Add file → Upload files.
5. Upload EVERYTHING INSIDE the extracted folder, including:

   index.html
   styles.css
   voice-language.css
   v4.css
   app.js
   voice-language.js
   v4.js
   firebase-config.js
   cloud-sync.js
   firestore.rules
   sw.js
   manifest.webmanifest
   assets/
   icons/
   README.txt
   FIREBASE_SETUP_GUIDE.txt
   VIDEO_MEDIA_STATUS.txt
   VOICE_LANGUAGE_GUIDE.txt
   TEST_REPORT.txt
   SOURCES.txt

6. Do not upload only the ZIP file.
7. Commit directly to the main branch.
8. Confirm index.html appears directly on the repository’s main page—not inside another folder.
9. Open Settings → Pages.
10. Select Deploy from a branch → main → /(root), then Save.
11. Open the published address, usually:

   https://YOUR-USERNAME.github.io/fitness-v4/

12. Open the app online once before testing offline use.

UPDATING AN EXISTING FITNESS APP REPOSITORY
-------------------------------------------
1. Export a backup from the existing app first.
2. Upload all Fitness V4 files and replace matching files.
3. Delete obsolete V3-only files if GitHub still contains them, especially old assets/videos and assets/posters files that are not in the V4 package.
4. Keep the same website address if you want the app to migrate compatible local V2/V3 data automatically.
5. Open the updated app and use Settings → Check for update.
6. If the installed PWA still shows the older build, close it completely, reopen it while online, and check for update again.

FIREBASE
--------
The app works locally without Firebase. Automatic device-to-device sync requires the separate steps in FIREBASE_SETUP_GUIDE.txt. Do not leave Firestore in test mode for a public app.
